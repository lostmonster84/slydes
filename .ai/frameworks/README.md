# Universal Frameworks Directory

> **Reusable across all projects**
> This directory contains universal planning, content, and design frameworks that can be applied to any project.

## Frameworks

### APEX.md
**Meta-Framework** - All-Protocol EXecution (The Ultimate Orchestrator)

Use APEX for:
- Major features requiring full stack (backend + frontend + mobile + quality)
- Complete systems (blog, testimonials, products, team members)
- High-visibility pages requiring all frameworks
- When you want zero missed steps and production-ready output

**One Command:**
`APEX: [feature description]` → Complete production-ready system

**The 6-Stage Workflow:**
1. **RAPID** - Interpret rough input → comprehensive requirements
2. **CODA** - Plan comprehensively (Context → Objective → Details → Acceptance)
3. **CRUDX** - Build complete backend (Database → API → Admin UI → Integration)
4. **AIDA** - Structure content for conversion (Attention → Interest → Desire → Action)
5. **TOUCH** - Transform for mobile-native (iOS/Android patterns, 60fps)
6. **SOPHIA** - Audit quality (8 dimensions, target 85+)

**Result:** Production-ready system with complete backend, custom admin, conversion-optimized content, mobile-native experience, and quality-audited design.

**Modes:**
- **Full APEX:** All 6 stages (default)
- **APEX-STATIC:** Skip backend (CRUDX)
- **APEX-DESKTOP:** Skip mobile (TOUCH)
- **APEX-BACKEND:** Backend only
- **APEX-FRONTEND:** Frontend only

**See:** [APEX.md](./APEX.md) for full framework

### SOPHIA.md
**Design & UX Assessment Framework** - Sophisticated Objective Principles for Human Interface Assessment

Use SOPHIA for:
- Design quality audits
- UX evaluation
- Component assessment
- Design system compliance
- Objective design scoring

**8 Dimensions:**
1. **Typography** (0-15): Font sizing, hierarchy, consistency
2. **Spacing** (0-15): Padding, margins, scroll economy
3. **Touch Targets** (0-10): Button sizes, mobile usability
4. **Information Density** (0-15): Content organization, scroll economy
5. **Visual Hierarchy** (0-15): Emphasis, color usage
6. **Sophistication** (0-10): Design polish, attention to detail
7. **Consistency** (0-10): Pattern reuse, design system adherence
8. **Accessibility** (0-10): WCAG compliance, keyboard nav

**Scoring:** Total 0-100 points
- 90-100: Exceptional
- 85-89: Sophisticated (target)
- 75-84: Good
- 65-74: Acceptable
- Below 65: Needs Work

**See:** [SOPHIA.md](./sophia.md) for full framework

### RAPID.md
**Prompt Engineering Framework** - **R**ough input → **A**mplify → **P**arse → **I**nfer → **D**eliver perfectly

Use RAPID for:
- Translating rough problem descriptions into comprehensive prompts
- Generating bulletproof, enterprise-grade AI prompts for Cursor IDE
- Bug fixes, UI/UX issues, performance investigations, large builds
- Consistent quality through baked-in standards and quality gates

**Quick Reference:**
- **Input Patterns:** "broken" → Bug Fix, "looks shit" → UI/UX Fix, "slow" → Performance
- **Project Detection:** Auto-detect Ancarraig, Native, WildTrax, FootballStays
- **Templates:** Bug Fix, UI/UX Fix, UI/UX Investigation, Large Build, Performance
- **Specialized:** Contrast Fix, Responsive Fix, Animation Fix, Depth & Polish
- **Standards:** Glassmorphism, 60fps Framer Motion, WCAG AA, Mobile-first

**Operating Principles:**
1. **Assume Competence** - Don't explain basics
2. **Extract, Don't Interrogate** - Max 3 questions, often zero
3. **Interpret Generously** - Expand rough input into comprehensive prompts
4. **Overkill By Default** - Every prompt includes validation, edge cases, quality gates
5. **Speak His Language** - Direct, technical, actionable

**See:** [RAPID.md](./rapid.md) for full framework

### CRUDX.md
**Full-Stack Content Management Framework** - CRUD eXtended

Use CRUDX for:
- Any content that needs management (testimonials, routes, offers, blog posts)
- Building complete backend + frontend systems
- When adding dynamic content to pages
- Automatic trigger: "Add [content] to page" or "CRUDX"

**6 Layers:**
1. **Database** - Supabase PostgreSQL schema (snake_case)
2. **Types** - TypeScript interfaces (camelCase)
3. **API** - REST endpoints with transform layer
4. **Admin UI** - Custom admin page (never Sanity Studio)
5. **Components** - Reusable admin components
6. **Integration** - Nav, preview links, status badges

**Architecture Rules:**
- ❌ Never use Sanity Studio
- ✅ Always build custom admin UI
- ✅ Supabase for storage
- ✅ Transform layer required (snake_case ↔ camelCase)
- ✅ Complete CRUD operations (Create, Read, Update, Delete)

**See:** [CRUDX.md](./CRUDX.md) for full framework

### DEMX.md
**Design Variation Framework** - Demo eXplorer (Rapid Design Variation System)

Use DEMX for:
- Exploring visual approaches for UI elements
- Redesigning existing components
- Deciding between layout options
- Any design decision where multiple approaches are valid

**Trigger:**
```
DEMX: [target element]
```

**The 5-Step Process:**
1. **Micro-Context Check** - Confirm target in 1-2 sentences
2. **Generate 5 Variations** - Dramatically different approaches
3. **Score with AIDA** - 0-10 per dimension, 40 total
4. **Build Demo Page** - Live, interactive at `/demo/[feature]-variations/`
5. **Recommend Winner** - Highest score + reasoning

**Result:** Live demo page with 5 interactive variations, AIDA scores, and clear recommendation.

**See:** [DEMX.md](./DEMX.md) for full framework

## Usage Workflow

### For New Features

1. **Plan with CODA**
   - Think through Context → Objective → Details → Acceptance
   - Present understanding conversationally (Light CODA)
   - Or write formal plan if needed (Heavy CODA)

2. **Structure Content with AIDA**
   - Plan content flow: Attention → Interest → Desire → Action
   - Score each section (target 80+)
   - Optimize weak sections

3. **Evaluate Design with SOPHIA**
   - Audit design across 8 dimensions
   - Score components (target 85+)
   - Fix violations, re-score

4. **Transform for Mobile with TOUCH** (if mobile-focused)
   - Audit current mobile experience
   - Implement iOS/Android patterns
   - Add gestures and haptic feedback
   - Optimize for touch interactions

5. **Iterate and Refine**
   - Use frameworks to guide improvements
   - Track scores over time
   - Continuous quality improvement

## Best Practices

1. **Use frameworks together** - CODA for planning, AIDA for content, SOPHIA for design, TOUCH for mobile, RAPID for prompts
2. **Score objectively** - Use AIDA and SOPHIA scoring systems
3. **Iterate based on scores** - Track improvements over time
4. **Don't overthink** - Light CODA (conversational) is default
5. **Document decisions** - Heavy CODA when needed for future reference
6. **Mobile-first when applicable** - Use TOUCH for mobile-native experiences
7. **Be rapid with RAPID** - Rough input → high-quality output
8. **Explore with DEMX** - One word → 5 variations → pick winner
