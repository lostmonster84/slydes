## APEX Modes

### Full APEX (Default)

**Command:** `APEX: [feature description]`

**Executes:** All 6 stages (RAPID â†’ CODA â†’ CRUDX â†’ AIDA â†’ TOUCH â†’ SOPHIA)

**Use for:** Major features requiring complete stack

---

### Partial APEX Modes

Skip certain stages based on feature type:

#### APEX-STATIC
**Command:** `APEX-STATIC: [feature]`
**Skips:** CRUDX (Stage 3)
**Use for:** Static content without backend management
**Example:** Landing page hero section, static about page

#### APEX-DESKTOP
**Command:** `APEX-DESKTOP: [feature]`
**Skips:** TOUCH (Stage 5)
**Use for:** Desktop-only features
**Example:** Complex dashboards, admin interfaces

#### APEX-BACKEND
**Command:** `APEX-BACKEND: [feature]`
**Skips:** AIDA (Stage 4), TOUCH (Stage 5), SOPHIA (Stage 6)
**Use for:** Backend-only systems (APIs, data processing)
**Example:** Data import pipeline, webhook handlers

#### APEX-FRONTEND
**Command:** `APEX-FRONTEND: [feature]`
**Skips:** CRUDX (Stage 3)
**Use for:** Frontend-only features using existing APIs
**Example:** New UI component, visualization widget
